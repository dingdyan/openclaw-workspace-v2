# 小红书爆款自动引流脚本 (Xiaohongshu Promotion Bot)

## 项目简介
此脚本专为推广 `baokuan.cc.cd` 到小红书相关爆款笔记评论区而开发。
已实现老板要求的四大反风控策略，确保马甲号稳定运行：
1. **随机休眠15-45分钟**：每个引流周期结束后，随机休眠指定时长，模拟真人作息。
2. **模拟滑屏延时**：在寻找笔记和阅读笔记时，通过 `page.mouse.wheel` 模拟真实用户的滑动行为，并带有随机停顿时间（1.5-4秒）。
3. **避免高频复制粘贴**：输入评论时，脚本采用**逐字键入**（模拟敲击键盘），每次按键延迟 50-200 毫秒，彻底规避复制粘贴检测。
4. **随机化引流话术**：内置了多套高质量推广文案，每次随机抽取一条进行发送，降低文本重复率。

## 环境依赖
由于环境已经配置完毕，系统依赖（包括 Playwright 及相关浏览器内核组件）已经通过 `playwright install-deps` 安装完成。

## 如何运行（老板请看这里）

**第一步：注入马甲号 Cookie**
脚本目前没有合法的马甲号登录状态，所以在运行时会被小红书安全策略拦截（例如跳转登录页或弹出验证码）。
请将老板提供的有效 `cookie` 转换为 JSON 格式，并覆盖到同级目录下的 `cookies.json` 文件中。
格式示例：
```json
[
  {
    "name": "web_session",
    "value": "xxxxxx",
    "domain": ".xiaohongshu.com",
    "path": "/",
    "httpOnly": true,
    "secure": true
  },
  ...
]
```

**第二步：启动脚本**
可以直接通过以下命令让脚本在后台持续运行：
```bash
nohup python3 xiaohongshu_bot.py > xhs_bot.log 2>&1 &
```
或者直接在前台运行以查看实时日志：
```bash
python3 xiaohongshu_bot.py
```

## 日志监控 (实时汇报进展)
脚本的所有操作（找到的笔记数、评论内容、滑屏动作、错误信息等）都会实时输出到控制台，并同步保存到 `xhs_bot.log` 文件中。可以通过以下命令随时查看运行状态和遇到的问题：
```bash
tail -f xhs_bot.log
```

## 注意事项（核心资产）
- 此脚本为核心资产，请在安全的网络环境下运行（尽量配合代理IP使用，防止单IP高频访问）。
- 如果发现日志中频繁出现 `Detected that we are NOT logged in`，说明 Cookie 已经失效或者被风控踢下线，请及时更新 `cookies.json`。
- 如果小红书前端 DOM 结构发生大的变更（例如输入框选择器改变），脚本会在日志中记录 `Could not find comment input field`，届时需要维护选择器参数。
